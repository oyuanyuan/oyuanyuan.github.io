生命周期概念Maven的生命周期就是为了对所有的都见过程进行抽象统一。生命周期包括了：项目的清理、初始化、编译、测试、打包、集成测试、验证、部署和站点生成等几乎所有的构建步骤。Maven生命周期是抽象的，这意味着生命周期本身不做任何实际的工作，具体工作都由对应的插件来做 生命周期详解Maven拥有三套相互独立的生命周期，分别是：clean，default，site。每个生命周期都包含一些阶段（phase），这些阶段是有顺序的，并且后面的阶段依赖前面的阶段。执行某个阶段的命令，会把前面依赖的阶段也都执行一遍 clean生命周期目的是清理项目，阶段如下： pre-clean：执行清理前需要完成的工作 clean：清理上一次构建生成的文件 post-clean：执行清理后需要完成的工作 default生命周期定了真正构建时所需要的执行所有步骤，阶段如下： validate initialize generate-sources process-sources 处理项目主资源文件 generate-resources process-resources compile 编译项目的源代码 process-classes generate-test-sources process-test-sources generate-test-resources process-test-res ...

Maven仓库

发表于 2016-09-12   |   分类于 Maven系列   |     |   阅读次数
仓库概念构件：Maven世界中，任何一个依赖、插件或者项目构建的输出，都可以称之为构件仓库：得益于坐标机制，任何Maven项目使用任何一个构件的方式完全相同。在此基础上，Maven可以在某一位置统一存储所有Maven项目共享构件，这个统一的位置就是仓库。 仓库的分类仓库分为两类：本地仓库和远程仓库，远程仓库分为中央仓库、私服、其他公共库 本地仓库：Maven利用坐标使用构件的时候，首先查找本地仓库，如果本地仓库没有，就会去找远程仓库，远程仓库没有就报错 中央仓库：Maven核心自带的远成仓库，包含了绝大部分的构件，默认的远程仓库，Mavan的安装文件自带了中央仓库的配置 本地仓库自定义本地仓库的位置:(在settings.xml中配置) 123<settings> <localRepository>d:/maven_repo</localRepository></settings> 本地仓库构件的来源： 从远程仓库下载到本地仓库 mvn clean install，执行这个命令，项目的构建输出文件会安装到本地仓库 私服私服是一种特殊的远程仓库，它是架设在局域网内的仓库服务，私服代理广域网上的远程仓库，供局域网内的Maven用户使用比较流行的Maven私服软件—–Nexus 使用私服的好处： 节省 ...
