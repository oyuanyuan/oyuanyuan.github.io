Maven安装和配置

发表于 2016-09-07   |   分类于 Maven系列   |  

文章是基于maven实战，此篇只是稍作总结，以便快速翻看

介绍

Maven是一个扩平台的项目管理平台，Apache的一个开源项目。Maven主要服务基于Java平台的项目构建、依赖管理和项目信息管理。
官网：http://maven.apache.org/

###安装
1、先安装jdk（最起码1.4以上）
2、windows在官网下载apache-maven-3.3.3-bin.zip，解压。
3、配置环境变量，在Path中加入解压路径到bin结束(例如：D:\apache-maven-3.3.3\bin)
4、cmd窗口，执行 mvn -v,看到maven版本信息就是成功了。

eclipse上maven插件 m2eclipse

POM编写

<project>： 根目录
<modelVersion>： POM模型版本，Maven2和Maven3，它只能是4.0.0
<artifactId>：Maven项目的唯一ID,一般是子项目名(模块名)，推荐用实际项目名作为前缀
<groupId>： 项目所在组，例如：com.googlecode.myapp，com.googlecode表示一个公司，myapp是实际项目名字
<version>： 当前项目的版本
<name>： 对用户更为友好的项目名称

###项目代码分布

  • pom.xml放在项目根目录下面
  • 一般会把项目主代码放在：src/main/java/目录下，项目主代码会被最终打包进构建中（例如jar）
  • 一般项目测试代码放在:src/test/java/目录下，只会测试时用到，不会被打包
  • 项目构建的所有输出都在target/目录中

###常用命令
mvn -v  查看版本
mvn clean   清理输出目录target/
mvn compile  编译Maven项目主代码，将主代码编译到target/classes目录
mvn test  编译Maven项目测试代码
mvn package  项目打包
mvn install  安装，生成jar，供别的项目使用

一般执行命令的时候都会先执行clean，可以写成一条命令，例如： mvn clean compile，mvn clean install

###maven-compiler-plugin插件
maven-compiler-plugin指定jdk版本和编码

使用该插件的理由：

  1. maven2默认jdk是1.3，maven3默认jdk是1.5，当你项目引用java新版本中的新特性，项目编译会报错，所以需要手动指定jdk编译版本。
  2. 项目中编码和系统编码不一致。
